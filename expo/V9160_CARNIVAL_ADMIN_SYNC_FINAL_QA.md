## Access control

Carnival synchronization remains **administrator-only**.

- The Settings shortcut is rendered only when `useAuth().isAdmin` is true.
- Direct navigation to `/carnival-sync` is blocked for non-admin users.
- The blocked route does not render the Carnival WebView or start any sync process.
- Current administrators remain the email addresses defined in `ADMIN_EMAILS` inside `state/AuthProvider.tsx`.

## Mobile Carnival sync flow

The Carnival path is separate from the Royal Caribbean/Celebrity ingestion sequence.

1. Open Carnival My Offers and Cruise Deals while preserving the authenticated Carnival session.
2. Discover every dynamic rate code from personalized URLs, `tgo`, VIFP payloads, visible action cards, and Shop Now/Search Cruises links.
3. Build the exact personalized Carnival search URL for each code using two guests, VIFP number, tier code, residence, locality, currency, and full `tgo` data.
4. Paginate every result page instead of stopping at the first eight cruises.
5. Convert Carnival per-person displayed prices into a two-person total for consistent value comparisons.
6. Retry zero-row Shop Now pages and preserve existing rows for visible offers that temporarily fail.
7. Read My Cruises and Profile pages for booked cruises, completed history, VIFP identity, cruise-day points, total cruises, and Players Club data.
8. Treat booked/upcoming and completed/history as independent authority lanes so one cannot erase the other.
9. Present a review screen before Apply Sync.

## Crash hardening

- Large WebView messages are rejected before JSON parsing.
- Large Carnival payloads remain browser-side and are handed off in chunks.
- Duplicate ingestion starts are ignored.
- iOS WebContent termination is handled safely.
- Android render-process exits are handled safely.
- JSON parsing is defensive.
- Existing Carnival data is preserved when a live run is incomplete.

## Brand and profile isolation

- Carnival rows use `cruiseSource: carnival`, `brand: carnival`, and `casinoProgram: playersClub`.
- Carnival Apply Sync manages only Carnival rows.
- Royal Caribbean and Celebrity offers, cruises, bookings, history, and loyalty are left untouched.
- Carnival filters show Carnival records; Royal and Celebrity filters exclude them.
- Each Easy Seas user profile retains separate VIFP and Players Club identity/points.
- Shared travel inventory remains available to both household profiles where the application’s shared-inventory rules apply.

## Downloadable browser extension

The browser extension generated by `lib/chromeExtension.ts` and the checked-in files under `assets/easy-seas-extension` are synchronized.

- Carnival helper loads before the general content script.
- Personalized rate codes and `tgo` are parsed dynamically.
- Every rate code is paginated.
- Carnival CSV output is explicitly Carnival/Players Club data.
- Booked cruises, completed history, and VIFP data are included when visible.
